Built-in Validation

Automatic validation based on column type

Column Type Automatic Validation
email Valid email format (xxx@xxx.xxx)
number Numbers only
currency Number format with separators
date Date format according to configuration
time Time format HH:mm
Tip: If validation fails, meta.hasErrors: true and meta.validationErrors[] will contain error details.

Required Fields & allowEmpty

Control which columns are mandatory

By default, GridSheet considers all columns mandatory (must be filled). Use allowEmpty: true to make a column optional. 

columns: {
    0: { name: 'username', type: 'text' }, // Required
    1: { 
        name: 'notes', 
        type: 'textarea', 
        allowEmpty: true // Optional: row considered "filled" even if empty
    }
}
Note: Checkbox and Readonly columns are automatically considered optional/filled.

preSave Callback

Called before data is sent to server

Use for custom validation. Return false to cancel save.

endpoints: {
    save: {
        endpoint: 'api/save.php',
        preSave: function(payload) {
            // Consistent payload structure:
            // payload.operation = 'insert' | 'insert_batch'
            // payload.data = { row: {...} } or { rows: [...] }
            // payload.meta = { timestamp, hasErrors, validationErrors, ... }
            
            // Check built-in validation
            if (payload.meta.hasErrors) {
                console.log('Errors:', payload.meta.validationErrors);
                return false;  // Cancel save
            }
            
            // Custom validation - check salary in row data
            const salary = payload.data?.row?.salary;
            if (salary && parseFloat(salary) < 1000000) {
                alert('Minimum salary $ 1.000.000');
                return false;  // Cancel save
            }
            
            // Return true or undefined to proceed with save
        }
    },
    update: {
        endpoint: 'api/update.php',
        preSave: function(payload) {
            // payload.operation = 'update' | 'update_batch'
            // payload.data = { id, fields: {...} } or { rows: [...] }
            
            if (payload.meta.hasErrors) {
                return false;
            }
            
            // Access field values via payload.data.fields
            const salary = payload.data?.fields?.salary;
            if (salary && parseFloat(salary) < 1000000) {
                alert('Minimum salary $ 1.000.000');
                return false;
            }
        }
    }
}

Payload Structure

Consistent structure across all operations

Core Structure

// All payloads follow this structure:
{
    operation: 'insert' | 'insert_batch' | 'update' | 'update_batch' | 'delete' | 'batch',
    data: { ... },    // Operation-specific data
    meta: { ... }     // Metadata including validation
}

Meta Properties (Common)

Property Type Description
timestamp Number Operation timestamp (ms)
hasErrors Boolean True if validation failed
validationErrors Array List of validation errors (empty if none)
rowCount Number Number of rows affected

Contextual Properties

Property Available In Description
tempId insert Temporary ID for new row
fieldCount insert, update, batch Number of fields
rowIndex insert, update, delete Visual row index (for scrolling)
colIndex update (single cell) Column index
insertCount batch Number of rows to insert
updateCount batch Number of rows to update
deleteCount batch Number of rows to delete

Example Payloads

// INSERT (new row)
{
    operation: 'insert',
    data: {
        row: { name: 'John', email: 'john@test.com' }
    },
    meta: {
        timestamp: 1704567890000,
        hasErrors: false,
        validationErrors: [],
        rowCount: 1,
        tempId: 'temp_123',
        fieldCount: 2,
        rowIndex: 5
    }
}

// UPDATE (single cell)
{
    operation: 'update',
    data: {
        id: '123',
        fields: { salary: 120000 }
    },
    meta: {
        timestamp: 1704567890000,
        hasErrors: false,
        validationErrors: [],
        rowCount: 1,
        fieldCount: 1,
        rowIndex: 5,
        colIndex: 3
    }
}

// BATCH (Save All)
{
    operation: 'batch',
    data: {
        insert: [{ _tempId: 'temp_1', ... }],
        update: [{ id: '5', fields: {...} }],
        delete: ['10', '11']
    },
    meta: {
        timestamp: 1704567890000,
        hasErrors: false,
        validationErrors: [],
        rowCount: 4,
        insertCount: 1,
        updateCount: 1,
        deleteCount: 2,
        fieldCount: 5
    }
}

Validation Error Structure

// Each item in validationErrors[]
{
    rowId: 'temp_123',     // Row ID (temp or real)
    rowIndex: 2,           // Visual row index
    field: 'email',        // Field name
    colIndex: 3,           // Column index
    value: 'invalid',      // The invalid value
    type: 'email',         // Column data-type
    errorType: 'format'   // 'format' | 'empty' | 'type'
}

postSave Callback

Called after server response

endpoints: {
    save: {
        endpoint: 'api/save.php',
        postSave: function(result) {
            // result = response from server
            console.log('Server response:', result);
            
            if (result.success) {
                // Show success notification
                showToast('Data saved successfully!');
            } else {
                // Handle server error
                alert('Error: ' + result.message);
            }
        }
    }
}
Tip: Use postSave to update UI, refresh data, or show notifications.

Delete Confirmation

Use preDelete callback for delete operations

endpoints: {
    delete: {
        endpoint: 'api/delete.php',
        // preDelete receives rowId, not payload
        preDelete: function(rowId) {
            // Confirm delete
            if (!confirm('Delete row ' + rowId + '?')) {
                return false;  // Cancel delete
            }
        },
        postDelete: function(result, rowId) {
            console.log('Deleted row', rowId, ':', result);
        }
    }
}
Note: Delete uses preDelete/postDelete instead of preSave/postSave.

Full Example

Complete endpoint configuration with new payload structure

const gridsheet = new GridSheet({
    tableSelector: '#myTable',
    endpoints: {
        save: {
            endpoint: 'api/save.php',
            preSave: function(payload) {
                // Check validation errors
                if (payload.meta.hasErrors) {
                    console.log('Errors:', payload.meta.validationErrors);
                    return false;
                }
                // Custom validation on row data
                if (!payload.data.row?.name) {
                    alert('Name is required!');
                    return false;
                }
            },
            postSave: (result) => console.log('Saved:', result)
        },
        update: {
            endpoint: 'api/update.php',
            preSave: (payload) => !payload.meta.hasErrors,
            postSave: (result) => console.log('Updated:', result)
        },
        delete: {
            endpoint: 'api/delete.php',
            preDelete: (rowId) => confirm('Delete row ' + rowId + '?'),
            postDelete: (result, rowId) => console.log('Deleted:', rowId)
        },
        // Batch Mode only
        batch: {
            endpoint: 'api/batch.php',
            preSave: function(payload) {
                if (payload.meta.hasErrors) {
                    alert('Please fix errors first!');
                    return false;
                }
                return confirm(
                    'Save ' + payload.meta.insertCount + ' new, ' +
                    payload.meta.updateCount + ' updated, ' +
                    payload.meta.deleteCount + ' deleted?'
                );
            },
            postSave: (result) => console.log('Batch saved:', result)
        }
    }
});
