Guide to row operations and context menu configuration

Copy & Paste Selection Keyboard Context Menu

Context Menu Options

Right-click any cell or row to access common actions

GridSheet provides a built-in context menu with several key actions:

  • Copy: Copy selected cell content or selected rows.
  • Paste: Paste data from clipboard starting at selected cell.
  • Clear Row: Resets data in all cells of the selected row(s) but keeps the UI row intact.
  • Delete Row: Removes the row from the UI and triggers a delete operation.

Clear Row vs. Delete Row

Understanding the difference in data handling

Feature Clear Row Delete Row
UI Impact Cells become empty, row remains visible. Row is completely removed from table.
Direct Mode Triggers update endpoint with empty values. Triggers delete endpoint.
Batch Mode Marked as "edited" with empty values in localStore. Marked as "deleted" in localStore.
Pro Tip: Use Clear Row if you want to keep the row index and table structure but reset the data. Use Delete Row for permanent removal.

Configuration: allowDeleteRow

Restrict or enable row deletion

You can control whether the "Delete Row" option appears in the context menu using the allowDeleteRow option. 

const gridsheet = new GridSheet({
    tableSelector: '#myTable',
    allowDeleteRow: false // "Delete Row" will be hidden from context menu
});
Option Type Default Outcome
true Boolean Default Shows both "Clear Row" and "Delete Row".
false Boolean - Hides "Delete Row". ONLY "Clear Row" is visible.

Keyboard Shortcuts

Quick cell clearing using the keyboard

  • Delete or Backspace: Clears the currently selected cell(s) without removing the row.
