Quick Start

Minimal configuration to get started

JavaScript 
const gridsheet = new GridSheet({
    tableSelector: '#myTable',
    // Column Configuration
    columns: {
        0: { name: 'name', type: 'text' },
        1: { name: 'email', type: 'email' },
        2: { name: 'office', type: 'select', options: 'api/offices.php' }
    },
    endpoints: {
        save: { endpoint: 'api/save.php' },
        update: { endpoint: 'api/update.php' },
        delete: { endpoint: 'api/delete.php' },
        batch: { endpoint: 'api/batch.php' }
    }
});

Core Options

Main parameters that are required/frequently used

Parameter Type Default Description
tableSelector String - CSS selector for table element. Required.
columns Object null v1.0.0 Column config object. See Column Types.
allowedFields Array null Deprecated. Use columns instead. Auto-generated from columns.
enableColumnNumber Boolean false Show automatic row number column on the left.
allowAddEmptyRow Boolean false Automatically add empty row after saving a new row.
allowDeleteRow Boolean true Enable "Delete Row" option in context menu. If false, only "Clear Row" appears.

emptyTable Options

Configuration for table starting empty

Parameter Type Default Description
emptyTable.enabled Boolean false Enable empty table mode.
emptyTable.saveMode String 'direct' 'direct' = save directly to server, 'batch' = save to localStorage first.
emptyTable.initialRows Number 1 Number of empty rows on initial load.
emptyTable.storageKey String 'gridsheet_data' Key for localStorage (batch mode only).
emptyTable.showSaveButton Boolean false Enable built-in Save button in batch mode.
emptyTable.showClearButton Boolean false Enable built-in Clear/Reset button in batch mode.
emptyTable.saveButtonText String 'Save' Text label for the built-in Save button.
emptyTable.autoRestore Boolean false Auto-restore pending data from localStorage on page load without showing confirmation modal (batch mode only).
Example: Batch Mode 
emptyTable: {
    enabled: true,
    saveMode: 'batch',
    initialRows: 10,
    storageKey: 'myapp_pending_data',
    showSaveButton: true,
    showClearButton: true,
    saveButtonText: 'Submit Data',
    autoRestore: true       // Auto-restore without modal dialog
}

Format Configuration

Formats are now defined inline per column in the columns config

Complete Example 
// v1.0.0 - Format is inline per column:
columns: {
    0: { name: 'salary', type: 'currency', 
         format: { thousandSeparator: '.', prefix: '$ ' } },
    1: { name: 'score', type: 'percentage', 
         format: { suffix: '%' } },
    2: { name: 'birthdate', type: 'date', 
         format: { format: 'DD/MM/YYYY' } }
}
Tip: See Formats page for more detail and interactive examples.

Footer Configuration

Add aggregate summary rows (sum, avg, count, min, max) at the bottom

Example 
footer: {
    enabled: true,
    label: 'Total'  // Label displayed in the first column
}

// Note: You must also set data-footer attribute on table headers
// <th data-name="qty" data-footer="sum">Qty</th>

dataTableOptions

Options passed to DataTables

Example 
dataTableOptions: {
    keys: true,          // Enable KeyTable (required for keyboard nav)
    paging: true,        // Enable pagination
    pageLength: 10,      // Rows per page
    order: [],            // Default sort (empty = no sort)
    scrollX: true,       // Horizontal scroll
    searching: true      // Search box
}

Language / i18n

Override specific labels or set the default language inline

Example 
// Override specific labels
language: {
    saveAll: 'Simpan Data',
    copyRow: 'Salin Baris',
    reset: 'Kosongkan'
}

// Or use predefined language object:
// language: GridSheetLang.id
Tip: See Language documentation for full list of translatable keys.

Select Options

Options are now defined inline per column in the columns config

Example 
// Options are inline per column:
columns: {
    0: { name: 'office', type: 'select', 
         options: 'api/offices.php' },       // Endpoint URL
    1: { name: 'status', type: 'select', 
         options: ['Active', 'Inactive'] }   // Static array
}

// Expected response from endpoint:
// ["New York", "Tokyo", "London", "Paris"]

extraHeaders

Custom headers for all requests (auth token, etc)

Example 
extraHeaders: {
    'Authorization': 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
    'X-CSRF-Token': 'abc123',
    'X-Custom-Header': 'value'
}

Full Configuration Example

Complete example with all options

Complete Example 
const gridsheet = new GridSheet({
    // Required
    tableSelector: '#myTable',
    
    // Core Options
    enableColumnNumber: true,
    allowAddEmptyRow: true,
    // Column Configuration
    columns: { 0: { name: 'name' }, 1: { name: 'email' } },  // See Column Types docs
    
    // Empty Table Mode
    emptyTable: {
        enabled: false,
        saveMode: 'direct',
        showSaveButton: true,
        showClearButton: true
    },
    
    // Footer Default Option
    footer: {
        enabled: true,
        label: 'Total'
    },

    // Language Setting / Override
    language: {
        saveAll: 'Submit Data'
    },
    
    // DataTables Options
    dataTableOptions: {
        keys: true,
        paging: true,
        pageLength: 10
    },
    
    // Auth Headers
    extraHeaders: {
        'Authorization': 'Bearer token'
    },
    
    // Endpoints
    endpoints: {
        save: {
            endpoint: 'api/save.php',
            preSave: function(payload) {
                if (!payload.name) return false;
            },
            postSave: function(result) {
                console.log('Saved:', result);
            }
        },
        update: {
            endpoint: 'api/update.php'
        },
        delete: {
            endpoint: 'api/delete.php'
        },
        batch: {
            endpoint: 'api/batch.php',
            preSave: function(payload) {
                console.log('Sending batch update...');
            },
            postSave: function(result) {
                alert('Batch update successful!');
            }
        }
    }
});
