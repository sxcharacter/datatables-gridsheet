Overview

GridSheet supports 4 endpoints with consistent payload structure

Endpoint Mode Function
save Direct Insert new row (single or batch)
update Direct Update cell/row (single or batch)
delete Direct Delete row
batch Batch All operations in one call (insert + update + delete)
Payload Structure: All requests use consistent format: { operation, data, meta }

Endpoint Configuration

endpoints: {
    // Direct Mode: Insert new row
    save: {
        endpoint: 'api/employees/save',
        preSave: function(payload) { /* validate */ return true; },
        postSave: function(response) { /* handle response */ }
    },
    // Direct Mode: Update cell/row
    update: {
        endpoint: 'api/employees/update',
        preSave: function(payload) { return true; },
        postSave: function(response) { }
    },
    // Direct Mode: Delete row
    delete: {
        endpoint: 'api/employees/delete',
        preDelete: function(rowId) { return confirm('Delete?'); },
        postDelete: function(response, rowId) { }
    },
    // Batch Mode: All operations in one call
    batch: {
        endpoint: 'api/employees/batch',
        preSave: function(summary) { return true; },
        postSave: function(response) { }
    }
}

Save Endpoint Direct Mode

Insert new row to database

POST Request Payload

{
    "operation": "insert",
    "data": {
        "row": {
            "name": "John Doe",
            "email": "john@example.com",
            "salary": "120000"
        }
    },
    "meta": {
        "tempId": "temp_1704567890_abc",
        "timestamp": 1704567890000
    }
}

POST Multiple Rows Insert (Paste/Fill Handle)

// operation: "insert_batch" - for multiple new rows
{
    "operation": "insert_batch",
    "data": {
        "rows": [
            { "_tempId": "temp_123", "name": "John", "email": "john@test.com" },
            { "_tempId": "temp_456", "name": "Jane", "email": "jane@test.com" }
        ]
    },
    "meta": {
        "rowCount": 2,
        "timestamp": 1704567890000
    }
}

Expected Response (Success)

CRITICAL for Inserts: Return the mapped database `id` for every `tempId` you inserted. GridSheet requires this mapping to update the table row's ID locally! 
{
    "status": "ok",
    "data": {
        "inserted": [
            { "tempId": "temp_123", "id": 100 },
            { "tempId": "temp_456", "id": 101 }
        ]
    }
}
When is insert_batch used?
- Paste multiple new rows (Ctrl+V on empty rows)
- Saves all new rows in a single API call instead of individual calls

Update Endpoint Direct Mode

Update cell or multiple fields

POST Single/Multiple Fields Update

// operation: "update" - for single row
{
    "operation": "update",
    "data": {
        "id": "123",
        "fields": {
            "name": "John Doe Updated",
            "salary": "55000000"
        }
    },
    "meta": {
        "fieldCount": 2,
        "timestamp": 1704567890000
    }
}

POST Multiple Rows Update (Paste/Fill Handle)

// operation: "update_batch" - for multiple rows
{
    "operation": "update_batch",
    "data": {
        "rows": [
            { "id": "123", "fields": { "salary": "55000000" } },
            { "id": "124", "fields": { "salary": "60000000" } }
        ]
    },
    "meta": {
        "rowCount": 2,
        "timestamp": 1704567890000
    }
}

Response

{
    "status": "ok",
    "data": {
        "updated": 2
    }
}
When is update_batch used?
- Paste multiple rows (Ctrl+V on multi-row selection)
- Fill handle drag across multiple rows

Batch Endpoint Batch Mode

All operations in a single API call (for Batch Mode)

Batch Mode Requirement: The endpoint handles inserts, updates, and deletes simultaneously. If new records are inserted locally, they must map standard DB ID's back to GridSheet's internal _tempId to track the rows moving forward.

POST Request Payload

{
    "operation": "batch",
    "data": {
        "insert": [
            { "_tempId": "temp_1708892_xyz", "name": "John", "email": "john@test.com" }
        ],
        "update": [
            { "id": "10", "fields": { "name": "Updated Name" } }
        ],
        "delete": ["20", "25"]
    },
    "meta": {
        "insertCount": 1,
        "updateCount": 1,
        "deleteCount": 2,
        "timestamp": 1704567890000
    }
}

Expected Response

CRITICAL for Inserts: You MUST return an array in inserted that maps the client-side tempId to the new server-generated database id. GridSheet uses this to magically swap the temporary HTML `tr id` to your DB's primary key! 
{
    "status": "ok",
    "data": {
        "inserted": [
            // Must match EXACTLY what was given by the frontend _tempId
            { "tempId": "temp_1708892_xyz", "id": 100 }
        ],
        "updated": 1,   // Just return the total count of success updates
        "deleted": 2    // Just return the total count of success deletions
    }
}

PHP Backend Example

Handle all operations with consistent payload

<?php
header('Content-Type: application/json');
$payload = json_decode(file_get_contents('php://input'), true);

$operation = $payload['operation'] ?? '';
$data = $payload['data'] ?? [];
$meta = $payload['meta'] ?? [];

switch ($operation) {
    case 'insert':
        // INSERT new row
        $row = $data['row'];
        // $stmt = $pdo->prepare("INSERT INTO ...");
        // $newId = $pdo->lastInsertId();
        echo json_encode(['status' => 'ok', 'data' => ['id' => $newId]]);
        break;

    case 'update':
        // UPDATE single row
        $id = $data['id'];
        $fields = $data['fields'];
        // UPDATE table SET ... WHERE id = $id
        echo json_encode(['status' => 'ok', 'data' => ['updated' => 1]]);
        break;

    case 'update_batch':
        // UPDATE multiple rows
        foreach ($data['rows'] as $row) {
            // UPDATE table SET ... WHERE id = $row['id']
        }
        echo json_encode(['status' => 'ok', 'data' => ['updated' => count($data['rows'])]]);
        break;

    case 'insert_batch':
        // INSERT multiple new rows
        $inserted = [];
        foreach ($data['rows'] as $row) {
            $tempId = $row['_tempId'];
            unset($row['_tempId']);
            // INSERT INTO table ...
            $inserted[] = ['tempId' => $tempId, 'id' => $newId];
        }
        echo json_encode(['status' => 'ok', 'data' => ['inserted' => $inserted]]);
        break;

    case 'delete':
        // DELETE row
        $id = $data['id'];
        // DELETE FROM table WHERE id = $id
        echo json_encode(['status' => 'ok', 'data' => ['deleted' => 1]]);
        break;

    case 'batch':
        // BATCH: insert + update + delete
        $inserted = []; $updatedCount = 0; $deletedCount = 0;
        
        foreach ($data['insert'] ?? [] as $row) {
            // INSERT and track tempId => newId
            $inserted[] = ['tempId' => $row['_tempId'], 'id' => $newId];
        }
        foreach ($data['update'] ?? [] as $update) {
            // UPDATE row
            $updatedCount++;
        }
        foreach ($data['delete'] ?? [] as $id) {
            // DELETE row
            $deletedCount++;
        }
        
        echo json_encode([
            'status' => 'ok',
            'data' => [
                'inserted' => $inserted,
                'updated' => $updatedCount,
                'deleted' => $deletedCount
            ]
        ]);
        break;
}
?>

Extra Headers

Add auth token or custom headers to all requests

extraHeaders: {
    'Authorization': 'Bearer your-jwt-token',
    'X-CSRF-Token': 'csrf-token',
    'X-Custom-Header': 'value'
}
Tip: Headers will be sent in all requests to endpoints (save, update, delete, batch, selectOptions).
